## Session 2026-05-20

**Fokus:** Part III — Individniveau. ESCO+ACA integration. Rekruttering og karriere.

**Analytiske beslutninger truffet:**

- ESCO defineret som *baseline* (rollenorm), ACA som *signal* (gab opad fra normen)
- Gabet opad fra ESCO-normen er rekrutteringens vigtigste signal — ikke match mod normen
- Translation demand og Accumulation Runway vægter 2× i scoring-modellen fordi de er selvforstærkende over tid. De øvrige fem dimensioner er statiske kapacitetsmål (1×). Automation vulnerability fratrækkes (−1×).
- Accumulation Runway operationaliseret som tre subdimensioner: feedback loop speed + knowledge externalizability (ESCO transversal skill-andel) + path dependency depth (skill adjacency i ESCO-ontologien)
- ESCO placeres som motorblok i rekrutteringskapitlet (sektion 3) — ikke som appendiks efter barometeret
- Sektion 3b ("Normen bevæger sig") tilføjet som bro mellem ESCO-baseline og ACA-signal — forankret i WEF Future of Jobs 2025
- Velocity asymmetry integreret i sektion 2 med erhvervskrimi-tone — ikke som dyb analyse
- Fem arketyper defineret med progression lav → høj ACA:
  1. Process Keeper (ACA: 12)
  2. Domain Expert (ACA: 31)
  3. Workflow Designer (ACA: 54)
  4. Data Translator (ACA: 74)
  5. Capability Architect (ACA: 91)
- Karrierekurver forklaret mekanisk (ikke blot listet): hvorfor kurven bøjer som den gør per arketype
- "Nordjysk tragikomik i enterprise-format: guldet ligger på lageret, men ingen har oprettet varenummeret" — bevares urørt

**Forkastede retninger:**
- ESCO som appendiks efter barometeret — for svagt analytisk
- Scorecard med ensartet vægtning — ikke forsvarligt (løst med 2×/1×/−1× model)
- Tidshorisonter (2031/2036) som selvstændigt afsnit med procentsatser — for McKinsey. Integreret som mekanisk argument i sektion 3b i stedet.
